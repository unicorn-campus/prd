> 본 문서는 교육 목적으로 작성된 가상의 MVP 기획 산출물임

# 문제 해결 방향성

## 1. Needs Statement

'포인트와 혜택을 매번 찾아다니는 게 귀찮은 바쁜 실속형 직장인은, 여러 화면을 일일이 뒤지지 않고도  
손해 보지 않기 위하여, 지금 나에게 필요한 정보를 내가 찾지 않아도 앱이 먼저 알아서 모아 보여주는  
경험이 필요하다.'

## 2. 상세 설명

### 고객유형
포인트와 혜택을 매번 찾아다니는 게 귀찮은 통신사 멤버십 가입자 중, 이번 MVP의 최우선 고객 그룹인  
**바쁜 실속형 직장인(25~39세)**을 가리킴  
출퇴근·점심시간처럼 짧은 틈에만 앱을 확인하고, 편의점·카페 같은 소액 결제에서 할인·적립을  
신경 쓰지만 매번 화면을 찾아보는 게 귀찮아 포기하는 경우가 잦은 사람들임  
(고객분석.md의 세그먼트1, 통신사 멤버십 가입자 중 규모가 가장 크고 앱 재방문으로 바로 연결되는 층)

### 목적
킹핀 문제(P1: 결제 순간, 정보가 흩어져 있어 실시간 확인 실패)의 근본원인인 "포인트·등급·혜택  
정보가 서로 다른 원장(정보를 저장하는 장부 시스템)에 나뉘어 있어, 지금 상황에 맞는 정보를 미리  
모아 보여주는 장치가 없다"는 점을 없애는 것을 목적으로 함  
여러 화면을 일일이 뒤지지 않고도, 결제 순간뿐 아니라 소멸 임박·매장 조건까지 손해 보지 않게  
되는 것이 최종 목적임(킹핀문제.md 4장의 연쇄 해결 예상 문제와 연결됨)

### 원하는 것
특정 화면이나 알림 기능이 아니라, "내가 정보를 찾아다니지 않아도, 앱이 지금 이 순간 나에게  
필요한 정보를 먼저 알아서 모아 보여준다"는 경험 자체를 의미함  
예를 들어 결제 대기줄에서는 지금 쓸 수 있는 혜택을, 소멸이 다가오면 그 사실을, 매장에 도착하기  
전에는 그 매장에서 실제로 적용되는 조건을 굳이 찾지 않아도 먼저 마주하게 되는 경험임  
이는 구체적인 화면 설계나 기술(솔루션)이 아니라, 앞으로 만들 모든 기능이 따라야 할 큰 방향성임

## 3. 검증 체크리스트

- [x] 기능이 아닌 경험으로 표현되었는가?  
  → "먼저 알아서 모아 보여주는 경험"으로 표현했으며, 특정 화면·버튼·알림 기능을 지정하지 않음
- [x] 하나의 문장으로 명확하게 표현되었는가?  
  → '<고객유형>는 <목적>을 위하여 <원하는 것>이 필요하다' 형식의 단일 문장으로 작성함
- [x] 킹핀 문제의 근본원인 해결과 연결되는가?  
  → 킹핀 문제(P1)의 근본원인인 "정보 통합·개인화 장치 부재"를 없애는 것을 목적 문구에 직접 반영함
- [x] 솔루션이 아닌 방향성을 제시하는가?  
  → "어떤 화면을 어떻게 만들지"가 아니라 "정보를 찾지 않아도 먼저 보여준다"는 방향만 제시함
- [x] 측정 가능한가?  
  → 비즈니스가치.md의 측정지표(확인까지 화면 전환 횟수, 소멸 방지율, 알림 후 실제 확인율)로  
    이 경험이 실제로 만들어졌는지 확인 가능함
