# 킹핀 문제 분석

> 용어 설명
> - 킹핀 문제: 볼링에서 스트라이크를 치려면 반드시 쓰러뜨려야 하는 5번 핀처럼, 이 문제 하나를 풀면 다른  
>   여러 문제도 함께 풀리는 가장 핵심적인 문제를 가리킴
> - 원장: 포인트·등급·혜택 같은 고객 정보를 실제로 저장해 두는 장부 역할의 시스템임

## 1. 검증된 문제 리스트

| 문제 ID | 문제 설명 | 중요도 평균 | 불편도 평균 | 출처 |
|---------|-----------|-------------|-------------|------|
| P1 | 결제 순간, 정보가 흩어져 있어 실시간 확인 실패 | 3.6 | 3.6 | 문제가설.md, 문제검증인터뷰결과취합.md |
| P2 | 유효기간 소멸을 뒤늦게 인지 | 4.2 | 3.8 | 문제가설.md, 문제검증인터뷰결과취합.md |
| P3 | 매장 현장에서 조건 불일치로 혜택을 놓치거나 거절당함 | 3.4 | 3.6 | 문제가설.md, 문제검증인터뷰결과취합.md |

인터뷰에서 확인된 중요도만 보면 P2가 가장 높으나, 아래 5가지 기준(2·3장)으로 더 깊이 들여다보면  
P1의 근본원인이 P2·P3의 근본원인과 같은 뿌리를 공유하고 있음이 확인됨(2장 참고)

## 2. 문제 간 인과관계

### 다이어그램

```
P1 (근본원인: 지금 상황에 맞는 정보를 먼저 모아 보여주는 장치가 없음)
    │
    ├──→ P2 (중간문제: 소멸 임박 정보도 먼저 눈에 띄게 보여주지 못해, 광고로 오인하고 뒤늦게 인지)
    │
    └──→ P3 (표면문제: 매장 조건도 미리 비교해서 보여주지 못해, 현장에서야 다르다는 것을 알게 됨)
```

### 근거

세 문제의 5WHY 분석(문제가설.md 2~4장)을 나란히 놓고 보면, 가장 깊은 단계(WHY4~5)에서 표현만  
다를 뿐 똑같은 이야기가 반복됨

| 문제 | 가장 깊은 단계에서 확인된 부족한 점(문제가설.md 원문) |
|------|------------------------------------------------------|
| P1 | "지금 상황(어느 매장, 언제)에 맞는 혜택을 미리 골라주는 개인화 장치까지는 필요 없다고 판단해왔음" |
| P2 | "고객별 중요도에 따라 다르게 보내는 개인화 발송 로직이 없음" |
| P3 | "여러 혜택 중 무엇이 유리한지 비교하고, 조건이 바뀌면 미리 알려주는 개인화 비교·안내 장치의  
      필요성이 우선순위 있게 다뤄지지 않았음" |

세 문제 모두 "고객이 지금 처한 상황(결제 대기줄, 소멸 시점, 매장 도착 전)에 맞춰 필요한 정보를  
스스로 찾아다니지 않아도 먼저 알아서 보여주는 장치가 없다"는 같은 빈틈에서 출발함  
다만 이 빈틈이 실제로 겉으로 드러나는 방식이 다름

- P1은 이 빈틈이 가장 직접적으로 드러난 문제임. 근본원인이 "원장(정보를 저장하는 장부)이 여러 곳에  
  나뉘어 있어 통합해서 보여줄 방법이 없다"는, 회사가 직접 손댈 수 있는 내부 시스템 문제로 구체적으로  
  확인됨(문제가설.md 근본원인 중요도 표에서 ★★★★★로 가장 높게 평가됨)
- P2는 같은 빈틈이 "소멸 임박 사실을 개인별로 다르게, 눈에 띄게 알려주지 못한다"는 형태로 나타남.  
  다만 P2는 여기에 더해 "안내와 광고 메시지가 뒤섞여 온다"는 별도의 커뮤니케이션 문제가 함께 겹쳐  
  있어, P1의 인프라만 해결한다고 완전히 풀리지는 않음
- P3은 같은 빈틈이 "매장 조건을 미리 비교해서 알려주지 못한다"는 형태로 나타남. 다만 P3은 여기에  
  더해 "앱과 매장 결제 단말기가 실시간으로 연동되지 않는다"는, 회사 혼자 힘으로는 풀기 어려운  
  가맹점(제휴 매장) 쪽 인프라 문제까지 얽혀 있어 P1보다 해결이 더 오래 걸림

즉 P1을 먼저 풀면, 포인트·등급·혜택을 한 곳에 모아 상황에 맞게 보여주는 기반이 만들어지고, 이  
기반 위에 소멸 임박 표시(P2 완화)나 매장 조건 사전 비교(P3 완화)를 얹기가 훨씬 쉬워짐. 반대로  
P2나 P3을 먼저 풀어도 P1의 통합 조회 기반 자체가 만들어지지는 않음

## 3. 5가지 기준 평가

| 문제 ID | 영향력 | 빈도 | 심각도 | 근본성 | 해결가능성 | 총점 |
|---------|:------:|:----:|:------:|:------:|:----------:|:----:|
| P1 | 4/5 | 5/5 | 3/5 | 5/5 | 4/5 | **21/25** |
| P2 | 2/5 | 3/5 | 5/5 | 2/5 | 4/5 | 16/25 |
| P3 | 2/5 | 3/5 | 3/5 | 2/5 | 2/5 | 12/25 |

### 평가 근거

**영향력(해결 시 다른 문제도 함께 해결되는가)**  
- P1: 통합·개인화 조회 기반이 만들어지면 P2(소멸 임박 강조 표시)와 P3(매장 조건 사전 비교)를  
  얹을 발판이 생김 → 4점  
- P2: 안내·광고 메시지 구분은 P2 자체 문제 해결에 집중된 조치로, P1·P3로 번지는 효과가 작음 → 2점  
- P3: 매장 결제 단말기 연동은 P3에 국한된 조치로, P1·P2로 번지는 효과가 작음 → 2점

**빈도(고객이 얼마나 자주 겪는가)**  
- P1: 결제할 때마다, 즉 매일 반복됨(최우선 고객 그룹인 바쁜 실속형 직장인 기준) → 5점  
- P2: 유효기간 소멸은 매달 반복되는 수준 → 3점  
- P3: 매장을 방문할 때마다 발생 가능하나 결제 빈도보다는 낮음 → 3점

**심각도(고객이 느끼는 고통의 크기)**  
- P2: 포인트가 실제로 사라진 뒤에야 알게 되는 실제 금전 손해이며, 감정 곡선의 최저점을 기록함  
  → 5점(세 문제 중 가장 높음)  
- P1: 매번 반복되는 좌절이 쌓여 재방문 의지를 낮추지만, 당장의 금전 손해는 아님 → 3점  
- P3: 현장에서의 당황스러움은 있으나 손해 규모는 P2보다 작음 → 3점

**근본성(다른 문제들의 원인이 되는가)**  
- P1: 근본원인(통합 조회 체계·개인화 장치 부재)이 P2·P3의 근본원인(개인화 발송 로직 부재,  
  개인화 비교 장치 부재)과 같은 뿌리를 공유하며, 세 문제 중 유일하게 회사가 내부적으로 직접  
  통제할 수 있는 시스템 문제로 구체화되어 있음 → 5점  
- P2: 근본원인 중 하나("회사가 소멸로 이득을 봐서 방치한다")는 인터뷰에서 5명 중 3명이 확신  
  없는 부분동의에 그쳐 근거가 약함 → 2점  
- P3: 근본원인(매장 결제 단말기 연동 부재)은 P3 고유의 문제이며 다른 문제의 원인으로 이어지지  
  않음 → 2점

**해결가능성(현실적으로 해결 가능한가)**  
- P1: 소프트웨어 아키텍트(아키)가 이미 캐시(자주 찾는 정보를 빠르게 꺼내 쓰도록 임시 저장해 두는  
  공간)와 원장을 분리해 둔 경험이 있어, 기존에 확보한 기술 기반 위에서 조회 화면을 통합하는  
  수준의 작업으로 추진 가능 → 4점  
- P2: 안내 문구·발송 방식을 광고와 다르게 바꾸는 수준의 조치로, 비교적 빠르게 적용 가능 → 4점  
- P3: 앱과 회사 밖 제휴 매장의 결제 단말기(포스)를 실시간으로 연동해야 해서, 회사 혼자 힘으로는  
  일정을 통제하기 어려움 → 2점

## 4. 킹핀 문제 선정

- **킹핀 문제**: P1. 결제 순간, 정보가 흩어져 있어 실시간 확인 실패
- **선정 근거**:  
  1) 5가지 기준 총점에서 21/25점으로 세 문제 중 가장 높음(P2 16점, P3 12점)  
  2) 세 문제의 5WHY 분석을 나란히 놓고 보면, 가장 깊은 단계에서 모두 "고객 상황에 맞춰 필요한  
     정보를 먼저 알아서 보여주는 장치가 없다"는 같은 빈틈에서 출발함을 확인함(2장). 이 빈틈이  
     회사가 직접 통제 가능한 내부 시스템 문제(원장 통합)로 가장 구체적으로 드러난 문제가 P1임  
  3) 최우선 고객 그룹(바쁜 실속형 직장인)이 결제할 때마다, 즉 가장 높은 빈도로 겪는 문제이며,  
     이 그룹의 여정 전 단계에 걸쳐 나타나 MVP 목표(포인트 한눈에 보기)와도 바로 연결됨  
  4) 이미 확보된 캐시·원장 이원화 기술 기반 위에서 추진 가능해 해결가능성도 상대적으로 높음  
- **연쇄 해결 예상 문제**:  
  - P2(유효기간 소멸 뒤늦은 인지): 통합 조회 화면에 소멸 임박 정보를 눈에 띄게 얹어 보여주면,  
    광고로 오인해 알림을 무시하더라도 앱을 열 때마다 자연스럽게 인지할 기회가 늘어남  
    (단, 안내와 광고 문구를 구분하는 조치는 별도로 병행해야 완전히 해소됨)  
  - P3(매장 현장 조건 불일치): 통합 조회 체계 위에 매장별 조건 비교 정보를 얹으면 매장 도착 전  
    미리 확인할 기반이 생김  
    (단, 매장 결제 단말기와의 실시간 연동은 별도 과제로 추가 추진이 필요함)
