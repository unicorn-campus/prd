# 유저플로우·이벤트스토밍 리뷰 반영 결과

`docs/plan/think/es/` 하위 puml 8개와 유저플로우매핑.md에 대한 8명 팀원 리뷰 결과와 반영 여부를  
정리함. 판단 기준은 (1) 규제 위반·오류 여부, (2) MVP 범위(빠르게 만들 수 있는 것) 적합성,  
(3) 예외처리 일관성, (4) 실제 구현 가능성 4가지를 우선 적용함.

## AI 기회 발굴 의견

마법사(AI엔지니어)의 리뷰 중 당장의 오류 수정이 아니라 향후 AI/개인화 확장 가능성을 짚은 의견을  
별도로 정리함. MVP는 규칙 기반으로 단순하게 유지하되(핵심솔루션.md 실현가능성 평가 결과 반영),  
아래 지점들은 데이터·이벤트 설계 시점부터 향후 AI 적용 여지를 남겨둠.

1. **04(소멸 임박 알림) 발송 정책 개인화 확장 여지**: MVP는 전 고객 동일한 배치 발송 정책이지만,  
   추천혜택사용됨/미사용됨 이벤트가 쌓이면 고객별 알림 반응률 기반으로 발송 시점·빈도를 개인화하는  
   모델 적용을 검토할 수 있음
2. **05(놓친 혜택 복구) 저관심 고객 판정의 학습데이터화**: 현재는 규칙(최근접속빈도, 알림반응률,  
   미확인누적횟수) 기반 판정이지만, 이 데이터 자체가 향후 이탈예측/재활성화 모델의 학습 피처로  
   전환될 수 있음
3. **06(개인화 추천) 규칙 매칭 → 추천모델로의 진화 경로**: MVP는 "최근 30일 최다 결제업종 1개  
   기준" 단순 규칙이지만, 04/06에 추가한 추천혜택사용됨/미사용됨 이벤트가 축적되면 이 데이터를  
   학습 데이터 원천으로 삼아 규칙 매칭을 추천 모델로 고도화하는 것을 다음 단계로 검토할 수 있음  
   (핵심솔루션.md에서 이미 "실현가능성이 가장 낮은 솔루션"으로 평가되어 있어, 초기에는 규칙  
   기반을 유지하고 데이터가 쌓인 뒤 모델 적용을 재검토하는 순서가 타당함)
