# 고객경험 단계 정의

## MVP 주제
포인트 한눈에 보기와 알아서 챙겨주는 혜택 도우미 (통신사 멤버십 앱)

## 고객유형
포인트와 혜택을 매번 찾아다니는 게 귀찮은 통신사 멤버십 가입자  
(내 포인트·등급·혜택을 한 번에 확인하고, 손해 보지 않게 챙기고 싶은 목표를 가진 고객)

> 참고: 아래 단계는 이번 MVP로 개선하기 이전, 고객이 지금 실제로 겪고 있는 현재 경험(있는 그대로의 모습)임  
> 이후 고객경험 조사와 여정 맵 작성의 기준(가로축)으로 활용함

## 고객경험 단계
```
혜택 필요 인식 -> 정보 탐색(앱 헤매기) -> 확인 또는 포기 -> 혜택 사용 시도 -> 뒤늦은 손해 인지 -> 체념 및 반복
```

## 단계별 설명

### 1단계. 혜택 필요 인식
- 주요 행동: 커피를 사거나 편의점, 영화관 등에서 결제하려는 순간, "여기서 할인이나 적립을 받을 수 있지  
  않을까?"라는 생각이 문득 떠오름. 특히 바쁜 실속형 직장인은 출퇴근길·점심시간처럼 짧은 틈에 이런 생각을 함
- 좋은 생각/느낌: "맞다, 나 여기 멤버십 혜택 있을지도 몰라" 하며 잠깐 기대감이 생김
- 주요 Pain Points: 결제 순간은 짧고 뒷사람 눈치도 보여서, 확인할지 말지 망설이는 시간 자체가 부담스러움

### 2단계. 정보 탐색(앱 헤매기)
- 주요 행동: 멤버십 앱을 열어 내 포인트가 얼마인지, 등급이 무엇인지, 이 매장에서 쓸 수 있는 혜택이  
  있는지 찾아보려고 여러 메뉴를 눌러보며 돌아다님
- 좋은 생각/느낌: 메뉴를 찾다가 우연히 몰랐던 혜택을 발견하면 "오, 이런 것도 있었네" 하며 반가움을 느낌
  (다만 매우 드문 경우임)
- 주요 Pain Points: 포인트·등급·혜택 정보가 서로 다른 화면에 흩어져 있어 원하는 정보를 찾기까지 여러 번  
  눌러야 함. 알뜰 살림형 중장년 고객은 메뉴 단계가 복잡하면 중간에 헤매다 포기하기도 함

### 3단계. 확인 또는 포기
- 주요 행동: 시간과 인내심이 남아있으면 끝까지 찾아 확인하지만, 대부분은 "귀찮다"는 생각에 확인을  
  중간에 그만두고 그냥 결제함. 어쩌다 확인러 유형은 애초에 앱을 열어볼 생각조차 하지 않음
- 좋은 생각/느낌: 어렵게라도 확인에 성공하면 "그래도 손해는 안 봤다"는 안도감을 느낌
- 주요 Pain Points: 확인에 실패하고 포기하는 경우가 훨씬 많음. "이럴 줄 알았으면 그냥 안 찾아볼걸"  
  하는 허탈함과 함께, 나만 손해 보는 것 같은 찜찜함이 남음

### 4단계. 혜택 사용 시도
- 주요 행동: 앱에서 확인한 혜택을 실제로 매장 직원에게 제시하거나 결제 화면에서 적용해 사용해봄
- 좋은 생각/느낌: 혜택이 정상적으로 적용되면 "역시 챙기길 잘했다"는 뿌듯함을 느낌
- 주요 Pain Points: 혜택 조건(등급 제한, 사용 가능 매장 등)을 미리 정확히 몰라서 현장에서 적용이  
  안 되는 경우가 있어 당황스러움. 등급 관리형 고객도 여러 혜택을 비교하느라 시간을 많이 씀

### 5단계. 뒤늦은 손해 인지
- 주요 행동: 한참 지난 뒤 우연히 앱을 열었다가, 포인트 유효기간이 지나 사라졌거나 등급 혜택을 아예  
  쓰지 못하고 넘긴 것을 뒤늦게 발견함
- 좋은 생각/느낌: (이 단계에서는 긍정적 느낌이 거의 없음. 다만 "이제라도 알게 돼서 다행"이라는  
  최소한의 안도감은 있음)
- 주요 Pain Points: "왜 진작 알려주지 않았지?"라는 억울함과 손해 본 느낌이 큼. 통신사 앱 알림은  
  광고로 여겨 평소에 확인하지 않았던 것을 후회함. 실제로 가입자가 받는 포인트의 59.3%가 쓰이지  
  못하고 사라지는 것으로 나타나, 이런 경험이 드물지 않음(시장조사.md 참고)

### 6단계. 체념 및 반복
- 주요 행동: "다음번엔 미리 챙겨야지" 다짐하지만, 평소 앱을 자주 열어보지 않는 습관은 바뀌지 않아  
  똑같은 과정(1~5단계)이 되풀이됨
- 좋은 생각/느낌: 그래도 "통신 요금은 어쨌든 내고 있으니 크게 손해는 아니다"라며 스스로를 다독임
- 주요 Pain Points: 문제를 알면서도 스스로 해결할 방법이 없다고 느껴 관리 자체를 포기함. 이 반복이  
  쌓이면 멤버십 앱에 대한 관심과 만족도, 재방문 의지가 점점 낮아짐

## 단계 도출 근거
- 고객분석.md의 JTBD(고객이 해결하고 싶은 일) "내 포인트·등급·혜택을 한 번에 확인하고, 손해 보지  
  않게 챙기고 싶다"를 기준으로, 실제 결제 상황에서 시작해 확인 실패와 뒤늦은 손해 인지로 이어지는  
  현재 경험 흐름을 그대로 반영함
- 고객분석.md 세그먼트 1(바쁜 실속형 직장인)의 "매번 화면을 찾아보는 게 귀찮아 그냥 포기하고  
  결제하는 경우가 많음" 행동 패턴을 2~3단계에 반영함
- 고객분석.md 세그먼트 3(어쩌다 확인러)의 "혜택이 있는 줄도 몰랐다" 반응을 3단계(포기)와  
  6단계(체념)에 반영함
- 고객분석.md 세그먼트 2(알뜰 살림형 중장년)의 "메뉴가 복잡하면 중간에 포기함" 행동을 2단계에 반영함
- MVP정의.md에서 밝힌 "포인트 조회 -> 사용처 찾기 -> 혜택 챙기기"가 하나로 묶여 있지 않은 현재  
  문제를 2~4단계의 여러 메뉴 이동 흐름으로 구체화함
- 시장조사.md의 "가입자 1인당 연평균 지급 포인트의 59.3%가 쓰이지 못한 채 사라짐"이라는 조사  
  결과를 5단계(뒤늦은 손해 인지)의 근거로 사용함
- 시장조사.md의 "통신사 앱 알림은 광고로 여겨 확인하지 않는 경향" 트렌드를 5~6단계의  
  Pain Points(알림을 믿지 않아 놓치는 문제)에 반영함
- 이후 이 6단계 흐름 중 특히 2~5단계(정보 탐색, 확인 또는 포기, 혜택 사용 시도, 뒤늦은 손해 인지)에서  
  발생하는 불편을 이번 MVP(한 화면 보기 + 놓친 혜택 알림)가 집중적으로 해소하는 대상으로 삼음
