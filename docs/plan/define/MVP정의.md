# MVP 정의

## MVP 주제
포인트 한눈에 보기와 알아서 챙겨주는 혜택 도우미 (통신사 멤버십 앱)

## 주제 설명
통신사 멤버십 고객이 자신의 포인트와 등급, 받을 수 있는 혜택을 한 화면에서 쉽게 확인하고,  
쓸 수 있는 곳과 놓치고 있는 혜택을 앱이 먼저 알려주는 서비스임.  
기존에 흩어져 있던 "포인트 조회 → 사용처 찾기 → 혜택 챙기기"를 하나의 흐름으로 묶어,  
"내 포인트가 얼마인지, 어디서 어떻게 쓰는지 모르겠다"는 고객의 가장 큰 답답함을 해소함.

> 용어 설명  
> - 멤버십: 통신사 고객에게 등급과 포인트를 주고, 제휴 매장에서 할인이나 적립 혜택을 제공하는 제도임  
> - 등급: 고객을 이용 실적에 따라 나눈 단계(예: 일반/우수/최우수). 등급이 높을수록 혜택이 많음  
> - 제휴 매장(가맹점): 멤버십 포인트나 할인을 사용할 수 있는 카페, 편의점, 영화관 등 협력 가게임

## MVP 후보 평가

평가는 4가지 기준을 각각 5점 만점으로 매김. 점수 기준은 아래와 같음.  
- 시장 잠재력: 시장이 얼마나 크고 빠르게 커지는지, 빈 자리가 있는지  
- 사용자 불편(Pain Points): 고객이 느끼는 불편이 얼마나 심하고 자주 겪는지, 대신할 방법이 없는지  
- 실현 가능성: 3~6개월 안에 만들 수 있는지 (기술 난이도, 필요한 사람·시간)  
- 경쟁 우위: 남들과 다른 점, 기존 방법보다 나은 점이 있는지  

| 후보 | 시장 잠재력 | 사용자 불편 | 실현 가능성 | 경쟁 우위 | 총점 |
|------|-----------|-------------|-----------|----------|------|
| 후보1. 포인트 한눈에 보기 + 혜택 도우미 | 4/5 | 5/5 | 5/5 | 4/5 | 18/20 |
| 후보2. 상담 없이 스스로 해결하는 포인트 셀프 처리 | 4/5 | 4/5 | 3/5 | 3/5 | 14/20 |
| 후보3. 등급 올리기 미션·게임형 참여 서비스 | 3/5 | 3/5 | 3/5 | 4/5 | 13/20 |
| 후보4. 제휴 매장 지도·주변 혜택 추천 | 4/5 | 3/5 | 3/5 | 3/5 | 13/20 |

### 후보별 평가 근거

**후보1. 포인트 한눈에 보기 + 혜택 도우미**  
- 시장 잠재력(4): 국내 통신 3사 멤버십 가입자는 수천만 명 규모로 이미 큰 시장임.  
  다만 완전히 새로운 시장을 여는 것은 아니어서 성장 여지는 "확대"보다 "개선"에 가까워 4점.  
- 사용자 불편(5): "내 포인트 얼마인지, 어디서 쓰는지 모르겠다"는 불만이 가장 흔하고 반복적임.  
  매달 혜택을 놓치는 손해가 실제로 발생하고, 지금은 여러 화면을 뒤져야 해서 대안이 마땅치 않음 → 5점.  
- 실현 가능성(5): 기존 이벤트스토밍(포인트 조회·혜택 조회 기능)이 이미 있어 이를 고객 화면으로 엮는 수준임.  
  새로 만들 핵심은 "한 화면에 모아 보여주기 + 놓친 혜택 알림"으로 3~6개월 내 충분히 가능 → 5점.  
- 경쟁 우위(4): 조회에 그치지 않고 "앱이 먼저 챙겨주는" 알림형 경험이 차별점임 → 4점.  

**후보2. 상담 없이 스스로 해결하는 포인트 셀프 처리**  
- 시장 잠재력(4): 상담 수요가 크고 상담 비용 절감 효과도 있어 가치가 큼.  
- 사용자 불편(4): 상담 연결까지 오래 기다리는 불편이 크지만, 후보1처럼 매일 겪는 문제는 아님 → 4점.  
- 실현 가능성(3): 포인트 취소·추가 같은 처리는 돈과 직결되어 검증·예외 처리 부담이 큼.  
  잘못 처리 시 피해가 커서 안전장치 개발에 시간이 더 필요 → 3점.  
- 경쟁 우위(3): 셀프 처리 자체는 다른 서비스에도 있어 차별성이 약함 → 3점.  

**후보3. 등급 올리기 미션·게임형 참여 서비스**  
- 시장 잠재력(3): 흥미 요소지만 핵심 수요라기보다 부가 기능에 가까움 → 3점.  
- 사용자 불편(3): "불편 해소"보다는 "재미 제공"이라 불편의 심각도는 낮음 → 3점.  
- 실현 가능성(3): 미션 규칙·보상 설계가 복잡하고 남용 방지 장치가 필요 → 3점.  
- 경쟁 우위(4): 게임형 참여는 차별화 포인트가 될 수 있음 → 4점.  

**후보4. 제휴 매장 지도·주변 혜택 추천**  
- 시장 잠재력(4): 위치 기반 혜택 추천은 성장성이 있음 → 4점.  
- 사용자 불편(3): "어디서 쓰지?"의 불편을 풀어주지만 지도·위치 정보 없이도 대안이 존재 → 3점.  
- 실현 가능성(3): 지도, 위치 정보, 실시간 매장 정보 연동이 필요해 범위가 넓어짐 → 3점.  
- 경쟁 우위(3): 지도형 혜택 추천은 이미 유사 서비스가 많음 → 3점.

## 선정 이유
후보1(포인트 한눈에 보기 + 혜택 도우미)이 18점으로 가장 높아 최종 선정함. 근거는 아래와 같음.  

1. 가장 아픈 문제를 해결(불편 5점)  
   - 고객이 매달 반복해서 겪는 "내 포인트·혜택을 모르겠다"는 문제를 정면으로 해결함.  
   - 혜택을 놓치는 것은 곧 고객의 실제 손해라 체감 가치가 큼.  

2. 3~6개월 내 만들 수 있음(실현 가능성 5점)  
   - 기존 "Loyalty 2.0" 이벤트스토밍에 포인트 조회·혜택 조회 흐름이 이미 설계되어 있음.  
   - MVP는 그 기능을 고객이 쓰기 쉬운 하나의 화면으로 엮고, "놓친 혜택 알림"만 새로 더하면 됨.  
   - 돈을 직접 움직이는 처리(후보2)보다 조회·안내 중심이라 위험이 낮고 개발 범위가 명확함.  

3. 기존 자료와 정합성이 높음  
   - 이벤트스토밍의 6개 영역 중 "포인트 정보 / 혜택 정보 / 고객 알림서비스"를 그대로 활용함.  
   - 포인트 변동 시 문자 안내(nMIMO) 흐름을 "놓친 혜택 알림"으로 자연스럽게 확장 가능함.  

4. 명확한 차별점  
   - 단순 조회를 넘어 "앱이 먼저 챙겨주는" 경험으로 기존 조회 화면과 차별화됨.  

> MVP는 최소 핵심에 집중함. 첫 버전은 "포인트·등급·혜택 한 화면 보기 + 놓친 혜택 알림"까지로 한정하고,  
> 셀프 처리(후보2)·게임형(후보3)·지도(후보4)는 이후 확장 과제로 남김.

## 목표 비즈니스 도메인
**통신사 멤버십 서비스 — 고객용 포인트·혜택 조회 및 알림 도메인**  

- 대상 고객: 통신사 멤버십에 가입한 개인 고객(멤버십 앱 사용자)  
- 핵심 범위:  
  - 내 포인트 잔액·등급·사용 내역을 한 화면에서 조회  
  - 내 등급에서 받을 수 있는 혜택 목록과 사용 가능한 곳 안내  
  - 아직 안 쓴 혜택·놓치고 있는 혜택을 앱이 먼저 알려주는 알림  
- 제외 범위(이번 MVP 아님):  
  - 포인트 셀프 취소·추가 등 금전 처리, 상담사 처리 흐름  
  - 제휴 매장 지도·위치 기반 추천, 게임형 미션 서비스  
- 목표 가치: 고객이 "내 포인트와 혜택을 쉽게 알고, 놓치지 않고 챙기게" 하여  
  멤버십 이용 만족도와 앱 재방문을 높임.
