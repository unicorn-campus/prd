## 2. 핵심 문제 1: 결제 순간, 정보가 흩어져 있어 실시간 확인 실패

### 현상문제
편의점·카페 계산대 줄처럼 짧은 시간 안에 포인트·등급·혜택을 확인하려 해도, 정보가 여러 화면에  
흩어져 있고 지금 이 상황에 맞는 혜택을 앱이 미리 골라 보여주지 않아, 시간 안에 다 찾지 못하고  
결국 확인을 포기함

### 5WHY 분석

| 단계 | 질문 및 응답 |
|:----:|-------------|
| **WHY1** | 왜 결제 대기줄에서 포인트·혜택 확인을 포기하는가? → 정보가 여러 화면에 흩어져 있어 찾는 데 시간이 오래 걸림(8초~40초 소요) |
| **WHY2** | 왜 정보가 여러 화면에 흩어져 있는가? → 포인트·등급·혜택 정보가 서로 다른 원장(정보를 저장하는 장부 시스템)에 나뉘어 저장되어 있어, 한 번에 모아 보여주는 통합 조회 체계가 없음 |
| **WHY3** | 왜 통합 조회 체계가 없는가? → 멤버십 앱은 그동안 투자 우선순위에서 밀려, 기본적인 조회 기능 유지에 머물러 있었음 |
| **WHY4** | 왜 멤버십 앱 투자가 후순위로 밀렸는가? → 멤버십은 직접 매출을 만드는 상품이 아닌 부가 서비스로 여겨져, 지금 상황(어느 매장, 언제)에 맞는 혜택을 미리 골라주는 개인화 장치까지는 필요 없다고 판단해왔음 |
| **WHY5** | 왜 개인화 장치가 필요 없다고 판단해왔는가? → 고객이 결제 대기줄처럼 극도로 짧은 시간 안에, 뒷사람 눈치를 보며 확인해야 한다는 실제 사용 상황을 설계에 충분히 반영하지 못했음 |

### 다층적 근본원인 검토 (4관점)

| 측면 | 검토 내용 |
|------|----------|
| **사용자 심리적** | 여러 화면을 오가며 정보를 찾는 과정 자체가 피로하게 느껴지고, 뒷사람 눈치에 조급함이 더해짐.  인터뷰 검증 결과 60%(3명)는 여기에 더해 평소 앱을 켜는 습관이나 화면 조작 숙련도 차이도 함께  작용한다고 답함(예: 조작에 서툰 고객은 시스템 문제보다 본인 탓으로 체감) |
| **제도/시스템적** | 포인트·등급·혜택 정보가 서로 다른 원장에 저장되어 실시간 통합 조회 체계 부재, 멤버십 앱 투자 후순위,  지금 상황에 맞는 혜택을 골라주는 개인화 장치 없음 |
| **맥락적** | 결제 대기줄이라는 극도의 시간 제약 상황(대부분 수십 초 이내에 결론이 나야 함) |
| **커뮤니케이션** | 해당 없음(이 문제는 정보 전달 방식보다 정보 저장·조회 구조의 문제에 가까움) |

### 핵심 근본원인 식별

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 사용자 심리적 | 정보 탐색 자체가 주는 피로·조급함과 개인별 습관·조작 숙련도 차이 | ★★★★☆ |
| 제도/시스템적 | 포인트·등급·혜택이 서로 다른 원장에 나뉘어 저장되어 통합 조회 체계가 없음 | ★★★★★ |
| 맥락적 | 결제 대기줄이라는 짧은 시간 제약 | ★★★★☆ |
| 커뮤니케이션 | 해당 없음 | ★☆☆☆☆ |

> 핵심 근본원인: **제도/시스템적** — 포인트·등급·혜택 정보가 서로 다른 원장에 나뉘어 저장되어 있어,  
> 이를 한 화면에 모아 보여주는 통합 조회 체계 자체가 없다는 점이 가장 근본적인 원인임  
> 다만 인터뷰 검증 결과 60%(3명)가 이 설명에 부분동의하며 본인의 습관·조작 숙련도 차이도 함께  
> 언급함. 즉 시스템 원인 한 가지만으로는 완전히 설명되지 않으며, 개인별 사용 습관 차이가 함께  
> 작용하는 문제로 이해해야 함

## 3. 핵심 문제 2: 유효기간 소멸을 뒤늦게 인지

### 현상문제
포인트·혜택이 유효기간을 넘겨 사라진 뒤에야 손해를 알게 됨. 소멸 예정 알림이 왔어도 광고로 여겨  
무시하는 패턴이 매달 반복됨

### 5WHY 분석

| 단계 | 질문 및 응답 |
|:----:|-------------|
| **WHY1** | 왜 포인트가 사라진 뒤에야 손해를 알게 되는가? → 소멸 예정 알림이 와도 광고로 여겨 확인하지 않고 넘어감 |
| **WHY2** | 왜 소멸 예정 알림을 광고로 여기고 무시하는가? → 소멸 안내와 판촉성 광고 메시지가 같은 방식(문구, 발송 채널)으로 뒤섞여 와서 구분이 안 됨 |
| **WHY3** | 왜 소멸 안내와 광고 메시지가 구분되지 않고 뒤섞이는가? → 안내 발송 기준이 법에서 정한 최소 요건만 충족하도록 설계되어 있어, 고객별 중요도에 따라 다르게 보내는 개인화 발송 로직이 없음 |
| **WHY4** | 왜 개인화 발송 로직이 만들어지지 않았는가? → 포인트가 소멸되면 회사 입장에서는 회계상 비용(부채)이 줄어드는 이득이 있어, 소멸을 적극적으로 막을 유인이 상대적으로 약했다는 가설이 있음(단, 인터뷰에서 확신도는 낮게 확인됨) |
| **WHY5** | 왜 소멸을 막을 유인이 약한 구조가 계속 유지되는가? → 소멸 관리를 법적 최소 요건 수준으로만 처리해온 관행이 오래 굳어져, 이를 바꿔야 한다는 문제의식이 조직 내에서 크게 제기되지 않았음 |

### 다층적 근본원인 검토 (4관점)

| 측면 | 검토 내용 |
|------|----------|
| **사용자 심리적** | 과거 스팸성 메시지 경험이 쌓여, 통신사 알림 전체를 광고로 여기는 학습된 불신이 자리잡음 |
| **제도/시스템적** | 법적 최소 요건만 충족하는 발송 정책, 포인트 소멸이 회계상 이득이 되는 구조(가설, 확신도 낮음), 개인화 차등 발송 로직 부재 |
| **맥락적** | 해당 없음 |
| **커뮤니케이션** | 광고성 메시지와 소멸 예정 같은 중요 안내가 문구·형식상 구분되지 않음(인터뷰에서 가장 이견 없이 확인된 사실) |

### 핵심 근본원인 식별

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 사용자 심리적 | 알림을 광고로 여기는 학습된 불신 | ★★★★☆ |
| 제도/시스템적 | 소멸이 회계상 이득이 되는 구조(가설)와 개인화 차등 발송 로직 부재 | ★★★★☆ |
| 맥락적 | 해당 없음 | ★☆☆☆☆ |
| 커뮤니케이션 | 광고와 중요 안내가 구분되지 않는 발송 방식 | ★★★★★ |

> 핵심 근본원인: **제도/시스템적 + 커뮤니케이션(공동)** — 소멸 안내와 광고성 메시지가 문구·형식상  
> 전혀 구분되지 않아 고객이 광고로 오인하고 넘어가는 것이 가장 직접적인 근본원인임. "회사가 소멸  
> 이득 때문에 의도적으로 방치한다"는 설명은 인터뷰에서 5명 중 3명(60%)이 확신 없는 부분동의에  
> 그쳐, 참고 가설 수준으로 비중을 낮추고 실제로 관찰되는 커뮤니케이션 원인의 비중을 높여 조정함

## 4. 핵심 문제 3: 매장 현장에서 조건 불일치로 혜택을 놓치거나 거절당함

### 현상문제
매장에 도착해서야 등급 제한·매장 제한 같은 조건이 앱에서 본 것과 다름을 알게 되거나, 여러 혜택 중  
어느 것이 유리한지 미리 비교하지 못해, 현장에서 혜택을 다 챙기지 못하고 거절당하거나 다시 확인해야 함

### 5WHY 분석

| 단계 | 질문 및 응답 |
|:----:|-------------|
| **WHY1** | 왜 매장에서 혜택을 다 챙기지 못하고 거절당하거나 재확인이 필요한가? → 매장 도착 전에는 이 매장에 실제 적용되는 등급·매장 조건을 미리 확인할 방법이 없음 |
| **WHY2** | 왜 매장 도착 전에 조건을 미리 확인할 방법이 없는가? → 앱에 표시되는 혜택 정보와 매장 결제 단말기(포스) 사이에 조건이 실시간으로 연동되어 있지 않음 |
| **WHY3** | 왜 앱과 매장 결제 단말기가 실시간으로 연동되지 않는가? → 혜택 조건 정보가 정산 주기(예: 월 단위) 단위로만 갱신되는 운영 방식이 관행으로 굳어져 있음 |
| **WHY4** | 왜 정산 주기 단위 갱신 관행이 계속 유지되는가? → 실시간 연동 체계를 새로 구축하는 대신, 일정 주기로 일괄 갱신하는 기존 운영 방식을 그대로 따라왔음 |
| **WHY5** | 왜 기존 운영 방식을 계속 따라왔는가? → 고객이 여러 혜택 중 무엇이 유리한지 비교하고, 조건이 바뀌면 미리 알려주는 개인화 비교·안내 장치의 필요성이 우선순위 있게 다뤄지지 않았음 |

### 다층적 근본원인 검토 (4관점)

| 측면 | 검토 내용 |
|------|----------|
| **사용자 심리적** | 등급·매장별 조건을 고객이 직접 기억하고 비교해야 하는 인지 부담 |
| **제도/시스템적** | 앱과 매장 결제 단말기(포스) 간 실시간 연동 부재, 정산 주기 단위 갱신 관행, 개인화 비교 장치 없음 |
| **맥락적** | 매장 도착 전에 조건을 사전 확인할 기회 자체가 부재 |
| **커뮤니케이션** | 조건 변경 사항이 고객에게 실시간으로 전달되지 않음 |

## 5. 선정 근거 상세

### 검증 결과 반영 우선순위
인터뷰 검증 결과, 세 문제의 해결 우선순위는 다음 순서로 참고함  
1순위 문제2(유효기간 소멸, 평균 중요도 4.2점, 실제 금전 손해) → 2순위 문제1(정보 분산, 최우선  
세그먼트 체감 매우 큼) → 3순위 문제3(매장 조건 불일치, 특정 세그먼트에 집중)

### 영향력
- 문제1·문제2는 팀원 5명 전원이 공통으로 지목했고, 유저저니맵의 페르소나(최우선 고객 세그먼트인  
  바쁜 실속형 직장인)가 겪는 여정 전 단계에 걸쳐 나타남. 인터뷰에서도 최우선 세그먼트(이수민, 서지훈)  
  가 두 문제 모두에서 높은 점수를 줘 영향력이 재확인됨
- 문제3은 4명이 지목했고, 유저저니맵 4단계(혜택 사용 시도)에서 실제 근거로 확인됨. 다만 인터뷰에서는  
  세그먼트4(등급 관리형 고객)에게 특히 집중되어, "혜택을 적극적으로 챙기려는 고객"을 잃을 위험으로  
  좁혀서 봐야 함이 확인됨

### 빈도
- 문제1은 결제할 때마다 반복되는 가장 높은 빈도의 문제임(관찰·체험 자료에서 8초~40초 탐색 반복 확인).  
  다만 세그먼트3(어쩌다 확인러)처럼 애초에 확인을 시도하지 않는 고객에게는 체감 빈도가 낮게 나타남
- 문제2는 매달 반복되는 문제이며, 실제로 가입자가 받는 포인트의 59.3%가 쓰이지 못하고 사라짐(시장조사  
  자료). 인터뷰 평균 중요도(4.2점)로도 가장 높게 재확인됨
- 문제3은 매장을 방문할 때마다 발생 가능하나, 결제 빈도보다는 낮음

### 심각도
- 문제2가 감정 곡선의 최저점(1점)을 기록해 심각도가 가장 높음. 확인 실패는 찜찜함에 그치지만,  
  포인트가 실제로 사라진 것을 아는 순간 억울함과 회사에 대한 불신으로 이어짐. 인터뷰에서도 평균  
  중요도 4.2점으로 세 문제 중 가장 심각하게 체감됨이 재확인됨
- 문제1은 매번 반복되는 좌절이 누적되어 앱 재방문 의지를 낮추는 구조적 심각도를 지님. 특히 최우선  
  세그먼트(바쁜 실속형 직장인)에게 매일 반복되는 급한 문제로 확인됨
- 문제3은 현장에서의 당황스러움과 재확인 부담을 주지만, 손해 규모는 문제2보다 작음. 다만 적극적으로  
  관리하는 고객(세그먼트4)일수록 느끼는 배신감은 매우 큼

## 6. 선정되지 않은 후보와 제외 사유

### 그룹4. 혜택이 있다는 사실 자체를 모름
- 지목 인원: 최초 1명(도그냥)으로, 다른 세 후보 대비 근거 강도가 약했음
- 이번 MVP의 최우선 고객 세그먼트(바쁜 실속형 직장인)는 이미 앱을 알고 쓰는 사용자이며, "혜택 존재  
  자체를 모른다"는 저빈도 이용자에게 해당하는 문제로, 최우선 세그먼트의 핵심 불편과는 결이 다르다고  
  판단해 제외함
- 근본원인도 제도/시스템적 관점 1개만 제시되어, 다른 후보들처럼 4관점에서 균형 있게 뒷받침되지 못함
- **[제외 사유 재검토 필요]** 문제검증 인터뷰에서 동일한 불편이 최윤서(세그먼트3)뿐 아니라  
  서지훈(세그먼트1, 최우선 고객 세그먼트)에게서도 나옴. "최우선 세그먼트는 해당 없음"이라는 기존  
  제외 근거가 약해진 상태이며, 완전 폐기가 아니라 차기 우선순위 검토에서 상위 후보로 다뤄야 함  
  (자세한 내용은 7장 참고)

## 7. 인터뷰에서 새로 발견된 불편 검토

문제검증 인터뷰 과정에서 기존 문제가설 3개(및 제외된 그룹4)에 없던 불편 3건이 새로 제기됨. 각각  
문제3·그룹4와의 유사성을 검토하고, 이번 핵심 문제 구성에 반영할지 판단함

### 1. 포인트 적립과 앱 반영 사이의 시차
- 제기자: 이수민(세그먼트1, 최우선 세그먼트) 1명
- 내용: 결제 직후 포인트가 앱에 바로 보이지 않아 "왜 안 쌓였지" 하는 불안감이 반복됨
- 유사성 검토: 문제3(매장 현장 조건 불일치)이나 그룹4(존재 자체를 모름)와는 성격이 다름. 이미 적립된  
  포인트를 "확인하는 시점"의 신뢰 문제로, 기존 세 문제·그룹4 어디에도 해당하지 않는 별개의 불편임
- 판단: 지목 인원이 1명으로 근거가 아직 약해 이번 핵심 문제 3개에는 포함하지 않음. 다만 앱에 대한  
  신뢰를 조금씩 깎아먹는 요인으로 확인된 만큼, 차기 문제가설 후보로 기록해 둠(신규 후보로 보류)

### 2. 혜택이나 포인트가 있다는 사실 자체를 모름
- 제기자: 최윤서(세그먼트3), 서지훈(세그먼트1, 최우선 세그먼트) 2명
- 내용: 문제1·2·3은 "내가 뭘 가지고 있는지 이미 아는 상태에서 못 챙기는 상황"을 가정하지만, 이  
  응답자들은 "가지고 있는 것 자체를 모른다"고 답함
- 유사성 검토: 6장에서 제외했던 그룹4(혜택 존재 자체를 모름)와 내용이 동일함
- 판단: 그룹4를 제외할 때 근거로 삼았던 "최우선 세그먼트는 해당 없음"이 서지훈의 응답으로 약해짐.  
  이번 3개 핵심 문제 구성과 5WHY 분석은 그대로 유지하되, 그룹4를 "완전 보류"에서 "차기 우선순위  
  검토 시 최상위로 다뤄야 할 후보"로 격상함(6장 반영)

### 3. 앱 화면 조작 자체의 어려움
- 제기자: 이순자(세그먼트2) 1명
- 내용: "포인트·등급·혜택이 따로 저장돼 있어서 못 찾는다"는 문제1의 설명과 별개로, "그 전에 어느  
  메뉴를 눌러야 하는지부터 헷갈린다"는 점을 응답자 스스로 명확히 구분해서 지적함
- 유사성 검토: 문제1(정보 분산)과 인접하지만 응답자 본인이 "정보가 흩어진 문제와는 별개"라고  
  구분했고, 문제3·그룹4와도 성격이 다른 별개의 진입장벽임
- 판단: 최우선 세그먼트(세그먼트1)에서는 나오지 않고 세그먼트2(알뜰 살림형 중장년)에 국한된  
  응답이며 지목 인원도 1명이라, 이번 핵심 문제 3개에는 포함하지 않고 세그먼트2向 후속 검토  
  후보로만 기록함

### 종합 판단
신규 불편 3건 중 "혜택 존재 자체를 모름"만 기존 그룹4와 동일한 사안으로 확인되어 6장 제외 사유를  
수정함. 나머지 2건(적립 반영 시차, 앱 조작 난이도)은 근거(지목 인원)가 아직 약해 이번 핵심 문제  
3개 구성에는 반영하지 않고, 차기 문제검증 라운드의 후보로 기록만 해 둠
